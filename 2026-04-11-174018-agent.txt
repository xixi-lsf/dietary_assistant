  ---
  memory.py 讲解

  核心逻辑：三个函数

  _replay_weights(feedback_history) — 重放权重历史

  把用户所有历史反馈按时间顺序"重放"一遍，模拟每次反馈后口味权重的变化：

  每条反馈 → 提取评分 score（1-5）
           → delta = (score - 3) / 2.0  # 范围 -1 ~ +1
           → 对每个标签：
               new_weight = old * 0.9 + (0.5 + delta*0.3) * 0.1

  衰减系数 DECAY=0.9 意味着旧记忆保留90%，新反馈只影响10%，防止单次反馈剧烈改变
  偏好。每步记录一个快照（权重字典 + 方差）。

  _convergence_index(snapshots) — 收敛指数

  比较"最近5次"和"最早5次"的权重变化幅度：
  收敛指数 = 1 - (recent变化量 / early变化量)
  接近1说明权重已经稳定，系统"认识"了你的口味。

  GET /memory/stats — 统计接口

  组装并返回：
  - taste_weight_history — 每步快照（给折线图用）
  - score_trend — 每条反馈的评分 + 3条滑动平均
  - convergence_index — 收敛指数
  - current_memory — 当前权重/禁忌/目标
  - ab_comparison — 普通模式 vs agent模式的平均分对比

  ---
  前端可视化：4张图

  入口在 memory_stats_screen.dart，调用 GET /memory/stats，用 fl_chart 库绘图。

  怎么进入这个页面： 在 app 底部导航找到对应 tab，或从设置/记忆入口进入
  MemoryStatsScreen。

  ---
  图1：口味权重收敛曲线（_weightChart）

  - X轴：反馈次数（第几条反馈）
  - Y轴：权重值 0~1
  - 每条彩色折线 = 一个口味标签（如"偏辣"、"清淡"）
  - 点击折线显示 tooltip：标签名 + 当前权重值

  反映什么： 各口味标签的权重随时间如何变化。折线趋于平稳 →
  系统已稳定认识你的偏好；剧烈波动 → 偏好还在学习中。

  ---
  图2：权重方差曲线（_varianceChart）

  - X轴：反馈次数
  - Y轴：所有标签权重的方差
  - 橙红色折线 + 半透明填充

  反映什么： 方差大说明各标签权重差异大（口味复杂/分散）；方差趋近于0说明权重趋
  于一致（偏好集中或数据不足）。配合收敛指数一起看。

  ---
  图3：推荐满意度趋势（_scoreChart）

  - X轴：反馈次数
  - Y轴：评分 1~5
  - 蓝色半透明散点 = 每次原始评分
  - 蓝色实线 = 3条滑动平均

  反映什么： 随着记忆系统积累反馈，推荐质量是否在提升。滑动平均线上升 →
  系统越来越懂你；平稳或下降 → 推荐质量没有改善。

  ---
  图4：当前长期记忆卡片（_currentMemoryCard）

  不是折线图，是静态卡片：
  - 口味权重：每个标签一条进度条，绿色（≥0.7强偏好）/ 橙色（中等）/
  红色（≤0.3不喜欢）
  - 硬约束：红色 chip，绝对禁忌食材
  - 健康目标：绿色 chip
  - 偏好摘要：文字描述

  ---
  顶部摘要卡片

  收敛指数大数字（绿/橙/灰）+ 进度条 + 累计反馈条数 +
  文字摘要，一眼看出记忆系统的学习状态。

● 这些改进都可行，我逐条分析一下可行性和实现思路：

  ---
  1. 区分 agent / 普通模式的 memory ✅ 完全可行

  Feedback 表已经有 recommendation_mode 字段，ab_comparison
  也已经按模式分组了。问题是 UserMemory
  只有一份，没有按模式分开存权重。改进方向：_replay_weights 按模式过滤
  feedback_history，分别重放两条权重演化曲线，前端叠加对比显示。

  ---
  2. "重新推荐"纳入评估 ✅ 可行，但需要先埋点

  目前 Feedback 表没有记录用户是否点了"重新推荐"。需要：后端 Feedback 模型加一个
   requested_retry: bool
  字段，前端推荐页在用户点"重新推荐"时提交一条隐式反馈（score
  可以默认2分或单独处理）。然后统计中加入"重推率"指标。

  ---
  3. 横坐标出现连续"1"的问题 ✅ 是 bug，可以修

  原因是 fl_chart 的 getTitlesWidget
  对每个像素点都调用，没有做整数过滤。修复：加 interval 参数动态计算步长，或在
  getTitlesWidget 里判断 v == v.roundToDouble()
  才显示。横坐标压缩也好做，根据数据点数量动态设 interval = max(1, total/6)。

  ---
  4. 展示两种模式对比 ✅ 可行

  后端 ab_comparison 已经有数据了，只是前端没有渲染这部分。加一个柱状图或对比卡
  片即可，显示两种模式的平均分、方差、重推率。

  ---
  5. 口味标签归一化 ✅ 最有价值，也最复杂

  这是核心问题。现在标签是原始文本，"太油"、"油腻"、"偏油"会变成三个不同
  key，导致权重分散、图线混乱。

  改进方案：在 extract_feedback_tags 里让 AI 做两件事：
  1. 把负面描述转成正向标签（"太油" → "少油腻"，"太咸" → "口味清淡"）
  2. 归一化到预定义标签集（约20个标准标签，如"偏辣"、"少油"、"清淡"、"鲜甜"等）

  这样标签数量稳定，权重图线也会更干净。同样的逻辑可以用在用户 profile 的
  preferences 字段清洗上。
